Module bitset

A simple bitset implementation, accelerated with C.

Uses 32-bit integers for hypothetical portability. Tested and compatible with LuaJIT and Lua 5.1.

A bitset type.
Bitset:new (size, init)
Allocate a new bitset. Capable of allocating a new bitset, with an optional bit capacity. If specified, the bits of the newly allocated bitset can be set to all one. If called with a bitset as the first argument instead of a number, the bitset will be cloned.

Parameters:

  • size num or Bitset the size of a bitset to allocate. Or, if a bitset, the bitset to copy.
  • init bool if true, set all newly allocated bits.

Returns:

    Bitset a newly allocated bitset.
Bitset:set (idx)
Sets a single bit in the bitset, at a given index. The bitset is modified in place, but for convenience, it is also returned.

Parameters:

  • idx num the index of the bit to set.

Returns:

    Bitset the modified bitset.
Bitset:set_range (lo, hi)
Sets a range of bits in the bitset, [lo, hi). The bitset is modified in place, but for convenience, it is also returned. The parameters are named lo and hi, but if lo is greater than hi, set_range will swap them internally. The low bound is inclusive while the high bound is exclusive.

Parameters:

  • lo num the low index of the range to set.
  • hi num the high index of the range to set.

Returns:

    Bitset the modified bitset.

Bitset:intersection (lhs, rhs)
The intersection of two bitsets. Also available as the * operator. NOTE: Bitset:intersection is a constructive operation. That is, it allocates a new bitset to hold the results. For an in-place intersection, see Bitset:intersection_mut.

Parameters:

  • lhs Bitset the left-hand bitset to intersect.
  • rhs Bitset the right-hand bitset to intersect.

Returns:

    Bitset a newly allocated bitset containing the intersection.

See also:

    Bitset:intersection_mut
Bitset:intersection_mut (lhs, rhs)
The in-place intersection of two bitsets. NOTE: Bitset:intersection_mut is a destructive operation. That is, it mutates the left-hand operand passed to it instead of allocating a new bitset. For a constructive intersection operation, see Bitset:intersection.

Parameters:

  • lhs Bitset the left-hand bitset to intersect. Is mutated to contain the intersection.
  • rhs Bitset the right-hand bitset to intersect.

Returns:

    Bitset the left-hand bitset is modified in-place, but returned for convenience.

See also:

    Bitset:intersection

Bitset:difference (lhs, rhs)
The asymmetric difference of two bitsets. Also available as the - operator. Returns a newly allocated bitset with bits set iff they are set in the left-hand operand but not in the right-hand operand.

NOTE: Bitset:difference is a constructive operation. That is, it allocates a new bitset to return the results. For an in-place asymmetric difference operation, see Bitset:difference_mut.

Parameters:

  • lhs Bitset the source bitset.
  • rhs Bitset the bitset to "subtract" from the source.

Returns:

    Bitset a newly allocated bitset with all bits set that are set in lhs but not in rhs.

See also:

    Bitset:difference_mut
Bitset:difference_mut (lhs, rhs)
The in-place asymmetric difference of two bitsets. Clears any bits in the left-hand operand if they are set in the right-hand.

NOTE: Bitset:difference_mut is a destructive operation. That is, it modifies the left-hand operand in-place instead of allocating a new bitset to hold the result. For a constructive asymmetric difference operation, see Bitset:difference.

Parameters:

  • lhs Bitset the bitset to modify.
  • rhs Bitset the bitset to "subtract" from the left-hand.

Returns:

    Bitset the left-hand bitset is modified in place, but returned for convenience.

See also:

    Bitset:difference
Bitset:symmetric_diff (lhs, rhs)
The symmetric difference of two bitsets. Returns a newly allocated bitset with bits set iff they are set in only the left-hand operand or the right-hand operand, but not both.

NOTE: Bitset:symmetric_diff is a constructive operation. That is, it allocates a new bitset to hold the result rather than modifying the operands in-place. For an in-place symmetric difference operation, see Bitset:symmetric_diff_mut.

Parameters:

  • lhs Bitset the left-hand bitset for the symmetric difference operation.
  • rhs Bitset the right-hand bitset for the symmetric difference operation.

Returns:

    Bitset a newly allocated bitset with all bits set that are set in either lhs or rhs, but not in both.

See also:

    Bitset:symmetric_diff_mut
Bitset:symmetric_diff_mut (lhs, rhs)
The in-place symmetric difference of two bitsets. Modifies the left-hand operand to contain bits that are set in the right-hand but not in the left-hand, and clears any bits that are set in both.

NOTE: Bitset:symmetric_diff_mut is a destructive operation. That is, it modifies the left-hand operand in-place instead of allocating a new bitset to hold the result. For a constructive symmetric difference operation, see Bitset:symmetric_diff.

Parameters:

  • lhs Bitset the bitset to modify.
  • rhs Bitset the bitset to compare against.

Returns:

    Bitset the left-hand bitset is modified in place, but returned for convenience.
